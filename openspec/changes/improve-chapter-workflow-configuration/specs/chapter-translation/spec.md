## MODIFIED Requirements

### Requirement: 自动准备完整有序的章节文本

任务启动后，系统 SHALL 按当前项目页序处理全部页面；对于尚未具备完整 OCR 文本的页面，系统 MUST 复用现有检测和 OCR 能力补齐文本。只有全部页面准备成功后，系统才能按项目页序以及各页文本节点阅读顺序构造章节文本，其中每个文本块均具有可映射回 `PageId` 和 `NodeId` 的稳定任务内标识。若自动准备后仍存在空 OCR 文本块，系统 MUST 在翻译 API 调用前停止，展示具体页码、页面名称和文本块序号，并提供返回编辑器修复后重新发起任务的入口；没有文本节点的空白页不得被视为 OCR 缺失。

#### Scenario: 部分页面尚未识别

- **WHEN** 项目中存在缺少 OCR 文本的文本节点
- **THEN** 系统先处理这些未就绪页面，并在全部页面就绪后构造章节翻译输入

#### Scenario: 页面准备失败

- **WHEN** 任一页面的检测或 OCR 无法完成，或自动准备后仍有文本节点的 OCR 为空
- **THEN** 系统停止在翻译 API 调用之前，保留已有项目数据，显示失败页码、页面名称、文本块序号和原因，并允许返回编辑器修复后重新发起任务

#### Scenario: 没有文字的页面

- **WHEN** 页面不包含任何文本节点
- **THEN** 系统把该页面视为准备完成并保持其页序，不阻止后续翻译、修补和渲染
