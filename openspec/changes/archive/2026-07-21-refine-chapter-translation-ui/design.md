## Context

章节翻译页已经复用项目的 Card、Input、Switch 和 `LlmModelSelect`，本次需要保持现有设计系统与任务路径。模型收藏属于跨页面用户偏好，现有 `preferencesStore` 已用 Zustand persist 保存字体收藏等配置，适合作为一致的持久化位置。

## Goals / Non-Goals

**Goals:**
- 用独立卡片和响应式间距建立清楚的配置层级。
- 复用现有共享模型选择器增加收藏能力，并让收藏结果稳定置顶。
- 复用 `koharu-config` 偏好存储，避免新增并行持久化机制。

**Non-Goals:**
- 不调整章节翻译任务参数、后端调度、进度状态或渲染流程。
- 不增加模型分组管理、收藏同步账号或跨设备同步。
- 不改变其他页面现有模型选择器的默认交互；收藏能力通过可选属性启用。

## Decisions

1. 页面结构拆为翻译设置 Card、分批设置 Card 和独立操作区。相比继续在单一卡片中增加分隔线，这种结构能以最小 CSS 改动建立明确层级，也便于分批字段按开关状态展开。
2. `LlmModelSelect` 接收可选收藏键集合与切换回调，在组件内部完成收藏优先的稳定排序。相比为章节页面复制选择器，该方案保留搜索、虚拟列表和样式的一致性。
3. `preferencesStore` 新增 `favoriteModels` 与 `toggleFavoriteModel`，并提升持久化版本。模型键沿用包含 kind、Provider 和 modelId 的稳定目标键，避免不同 Provider 的同名模型冲突。
4. 收藏按钮使用现有 lucide `StarIcon`，阻止点击事件冒泡；收藏变更不触发 `onChange`，选择列表保持打开。

## Risks / Trade-offs

- [收藏后列表即时重排可能使指针位置变化] → 收藏按钮位于固定行尾，操作完成后保留弹层并用填充星标反馈状态。
- [共享选择器改动影响现有工具栏] → 收藏属性保持可选，未传入时维持既有列表和交互。
- [历史偏好数据缺少新字段] → 迁移时补充空数组，并在读取处使用默认值防止旧数据异常。
