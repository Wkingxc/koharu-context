## Why

主页与章节翻译页目前各自保存模型选择，用户即使已在主页加载并通过配置切换了 LLM，进入章节翻译仍可能看到另一个默认模型，操作链路不连贯。

## What Changes

- 进入章节翻译设置时，优先继承主页当前已加载且可用于 API 翻译的 LLM 模型。
- 同步继承模型所属的 API Provider，并在需要时调整为该模型支持的目标语言。
- 主页没有可用的已加载 API 模型时，保留章节翻译原有的上次选择与首项回退逻辑。
- 初始化继承完成后，允许用户在章节翻译页手动改选模型。

## Capabilities

### New Capabilities
- `chapter-llm-inheritance`: 章节翻译初始化时继承主页当前 API LLM 的优先级与回退行为。

### Modified Capabilities

## Impact

影响章节翻译页面的模型初始化逻辑及组件测试；不修改后端翻译协议、配置预设结构或本地模型行为。
