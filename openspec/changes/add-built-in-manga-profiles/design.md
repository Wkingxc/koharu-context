## Context

处理配置通过 Zustand `persist` 保存在 `koharu-processing-profiles`。持久化状态会覆盖 store 初始值，因此可以用初始值为全新安装提供预设，同时自然保留已有安装的状态。

## Goals / Non-Goals

**Goals:**
- 精确固化当前“日漫”和“韩语”两套处理配置。
- 新安装默认选中日漫配置。
- 不覆盖已有用户配置。

**Non-Goals:**
- 不自动写入或共享 API Key。
- 不在启动时自动加载远端 LLM，以免尚未配置 Provider 时产生错误。
- 不把内置配置设计为不可修改或不可删除的系统项。

## Decisions

- 使用固定 ID 和固定内容构造两套初始配置，保证测试与持久化行为稳定。
- `persist` 版本保持不变。没有持久化状态时使用内置初始值；已有状态继续覆盖初始值，因此不会被注入重复配置。
- 将“日漫”置于列表第一项并设为初始 `activeProfileId`。当前应用的后端默认流水线已与日漫配置一致，用户也可从菜单重新点击以显式应用完整快照。

## Risks / Trade-offs

- [内置远端模型尚未配置] → 仅预选配置而不在启动时自动请求模型，用户完成 Provider/API Key 设置后可一键应用。
- [用户删除内置配置后重新出现] → 已持久化的空列表仍优先于初始值，因此删除结果会保留。
