# Chapter Translation Execution Feedback

### Requirement: 默认最大输出 Token
章节翻译配置页 SHALL 将最大输出 Token 的默认值设为 `32000`，并 SHALL 允许用户在发起任务前修改该值。

#### Scenario: 首次进入配置页
- **WHEN** 用户首次进入章节翻译配置页
- **THEN** 最大输出 Token 输入框显示 `32000`

### Requirement: 按处理语义展示阶段进度
章节翻译任务 SHALL 上报全章图片总数，以及准备、翻译、修补渲染三个阶段各自已经处理的图片数量；运行页 MUST 对准备和修补渲染阶段以 `已处理图片数 / 总图片数` 展示，并保证计数不超过总数且不会倒退。翻译阶段 MUST 以原子请求或批次状态表达，不得展示批内逐页进度。

#### Scenario: 准备图片
- **WHEN** 页面已具备或完成检测与 OCR 所需结果
- **THEN** 准备阶段的已处理图片数包含该页面，并显示为 `X / N`

#### Scenario: 等待整批翻译
- **WHEN** 一批页面已提交 LLM 但尚未返回
- **THEN** 翻译阶段显示活动等待状态；分批模式同时显示当前批次，不显示页数进度

#### Scenario: 完成一批翻译
- **WHEN** 第 N 批结构化译文校验并写回成功
- **THEN** 后端按该批页面范围增加已翻译图片数，运行页更新批次状态但不显示批内页数

#### Scenario: 逐页完成修补渲染
- **WHEN** 后处理管线逐页完成当前批次
- **THEN** 修补渲染阶段按已处理页面累计显示 `X / N`

### Requirement: 章节译文初始颜色统一为黑色
章节翻译写回每个文本块时 SHALL 将显式文字颜色设为不透明黑色 `[0, 0, 0, 255]`，同时 MUST 保留该文本块已有的字体、字号、效果、描边和对齐设置。

#### Scenario: 原文字色预测不同
- **WHEN** 不同文本块具有不同的预测原文字色并完成章节翻译
- **THEN** 所有章节译文均以黑色渲染，且用户之后可在原编辑器修改颜色
